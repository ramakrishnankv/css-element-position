CSS Position properties

Positioning methods for an element is specified by position property in CSS. Possible five values for position proerty are;

  • static
  • relative
  • absolute
  • fixed
  • sticky

Properties "left", "top", "right" and "bottom" decides the position of the element. "z-index" property allows the element to stack over the another.

Static

By default every elements are positioned static. The element's position will be accroding to the normal content flow. Static value can use at any stage when position to be overrided to default. Properties left, top, right and bottom are not affected by static positioned element.

  • Properties top, left, z-index won't have any impact for static positioned element.
  • No need to specify position: static as by default an element is static positioned.
  • Observe, when static positioned element changed to absolute positioned element the initial space occuppied by the element is cleared completely.

Relative

An element with position: relative; is positioned relative to its normal position. Setting the top, right, bottom, and left properties of a relative positioned element will cause it to be adjusted away from its normal position. Other content will not be adjusted to fit into any blank space left by the element.

Absolute

Is positioned relative to the nearest positioned ancestor (instead of positioned relative to the viewport, like fixed). However; if an absolute positioned element has no positioned ancestors, it uses the document body, and moves along with page scrolling.

Fixed

Is positioned relative to the viewport, which means it always remains in the same place even if the page is scrolled. The top, right, bottom, and left properties are used to position the element. A fixed element does not leave a gap in the page where it would normally have been located.

Sticky

is positioned based on the user's scroll position. A sticky element toggles between relative and fixed, depending on the scroll position. It is positioned relative until a given offset position is met in the viewport - then it "sticks" in place (like position:fixed).
